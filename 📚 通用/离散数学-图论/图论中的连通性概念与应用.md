# 图论中的连通性概念与应用

**核心概念定义**

**连通性**描述了图中各个顶点之间相互可达、可通信的能力。对于无向图，如果任意两点之间都存在路径，则该图是**连通图**。对于有向图，如果任意两点之间的双向路径都存在，则称为**强连通图**。

- **无向图** 连通性：每对顶点 $(u, v)$ 之间都存在通路。
- **有向图** 连通性：区分**强连通**（每对顶点互相可达）和弱连通（把边看作无向时连通）。

**详细说明**

- **连通分量**：在一个无向图中，极大连通子图称为连通分量；每个连通分量内部顶点互相可达。
- **割点与桥**：若删除某顶点或边后导致图不连通，则称该顶点为**割点**，该边为**桥（割边）**。
- **连通度**（connectivity）：最少需去除多少个顶点/边，才能使图变为非连通。

**关键公式/代码**

- 路径存在：对于无向图 $G=(V,E)$，如果对任意 $u,v \in V$，存在路径，$G$连通。
- 连通分量数量的判定可以用深度优先遍历 DFS/BFS 实现：

```python
# 用DFS找无向图连通分量数
def count_connected_components(graph):
    visited = set()
    count = 0
    def dfs(v):
        visited.add(v)
        for u in graph[v]:
            if u not in visited:
                dfs(u)
    for node in graph:
        if node not in visited:
            dfs(node)
            count += 1
    return count
```

**例题/应用**

1. 判断如下无向图是否连通：$V=\{a, b, c, d\}$，$E=\{(a,b), (b,c)\}$。解：$d$与其它顶点无路径，故不连通。
2. 网络设计：为保证任意两台服务器可通信，物理网络需保持连通。

**易混淆点**

- **强连通**只适用于有向图，==与“连通”（一般为无向图）不同==。
- 删除顶点vs删除边的连通度定义不同，忌混淆。
